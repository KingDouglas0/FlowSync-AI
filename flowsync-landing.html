Problem

African Cities Are Gridlocked.

Lagos. Nairobi. Accra. Every morning, millions of commuters lose hours to congestion that existing infrastructure cannot solve.

The assumption has always been that traffic optimization must be centralized, expensive, and dependent on stable connectivity. FlowSync breaks that assumption entirely.

Congestion is not a density problem. It is a coordination failure — intersections that react instead of think.

01 / SIGNAL FAILURE
Fixed timers ignore reality
Traffic lights run on rigid cycles set years ago, with no awareness of actual vehicle density or flow.
02 / NO COORDINATION
Intersections operate in isolation
Adjacent junctions cannot communicate. Green waves are impossible. Queues cascade uncontrolled across corridors.
03 / COST BARRIER
Western ITS is unaffordable
Traditional intelligent traffic systems cost $50K–$200K per junction, require stable internet, and depend on centralized servers that fail with the grid.
04 / EMERGENCY RISK
Ambulances trapped in traffic
Emergency response times are 40% longer during peak hours. Manual traffic officers cannot coordinate corridor clearance fast enough.
How it works

Four Steps.
Fully Autonomous.

01
Sense
Camera and IR sensors measure real-time queue lengths and vehicle arrival rates at every lane, every second.
ESP32-CAM + YOLOv8
02
Decide
Onboard edge processor runs the adaptive timing algorithm. Lanes sorted by density. Green time allocated proportionally. No cloud needed.
ESP32 — on-device logic
03
Coordinate
ESP-NOW mesh broadcasts congestion state to neighbouring nodes. Downstream junctions pre-adjust before vehicles arrive.
ESP-NOW + LoRa mesh
04
Adapt
Cycle completes — HIGH to MEDIUM to LOW priority. Model rescans. Process repeats. Each cycle smarter than the last.
Continuous learning loop
Features

Built for
African Infrastructure.

Autonomous intersection control
Each node runs its own decision loop independently. No master server, no single point of failure. Remove one node — the network keeps working.
Decentralized by design
Mesh network coordination
Nodes communicate peer-to-peer via ESP-NOW (200m) and LoRa (3km+). No router, no internet, no broker. Pure distributed intelligence.
Up to 3km range · <1ms latency
Solar-hybrid power
10W solar panel charges a LiPo battery pack. 20+ hours of operation during grid outages. Nodes survive extended power cuts that disable conventional systems.
Grid-independent · 20+ hr backup
Emergency corridor protocol
RF beacon or acoustic siren detection triggers a coordinated pre-cleared green wave across the entire corridor — not junction by junction — before the vehicle arrives.
Corridor-wide · Pre-emptive clearance
Live dashboard & API
Flask serves a real-time browser dashboard showing lane counts, active signals, cycle position, countdown timer, and full event log. REST API for ESP32 integration.
HTTP + JSON · Accessible on-network
Retrofit compatible
Mounts on existing signal poles. GPIO pins map directly to a relay board that drives standard 220V signals. Zero civil engineering. Zero road works. Same firmware.
Drop-in hardware upgrade
Live system

One Intersection.
Three Lanes.
Real Intelligence.

The adaptive timing engine sorts lanes by vehicle density every cycle — highest to lowest — and allocates green time proportionally. IR sensors on lanes B and C trigger a one-time 10-second extension when vehicles are detected mid-phase.

16+ vehicles → 70s green
6–15 vehicles → 40s green
0–5 vehicles → 25s green
